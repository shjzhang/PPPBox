57 // Model of the troposphere, used to compute non-dispersive delay of
58 // satellite signal as function of satellite elevation as seen at the
59 // receiver. Both wet and dry components are computed.
60 //
61 // The default model (implemented here) is a simple Black model.
62 //
63 // In this model (and many others), the wet and dry components are
64 // independent, the zenith delays depend only on the weather
65 // (temperature, pressure and humidity), and the mapping functions
66 // depend only on the elevation of the satellite as seen at the
67 // receiver. In general, this is not true; other models depend on,
68 // for example, latitude or day of year.
69 //
70 // Other models may be implemented by inheriting this class and
71 // redefining the virtual functions, and (perhaps) adding other
72 // 'set...()' routines as needed.
