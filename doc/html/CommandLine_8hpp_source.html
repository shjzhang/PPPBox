5 // NB. Repeatable means a parse error will be generated if the option appears more
6 // than once, unless that option is repeatable. Only vector types (2,5,7) can have
7 // more than one value, but this has nothing to do with being repeatable!
8 // NB. Options that go into vectors (types 2,5,7) may have arguments value,value,...
9 // and @files.lst where files.lst is a file containing values separated by spaces
10 // and/or linefeed and NOT starting with # (but not val,val,.. inside the file).
11 // NB. There is an issue for repeatable options that are not vector types, namely if
12 // there is more than one value, which to use: values[0] or values[values.size()-1]?
13 // NB. This is related to the question for bool options - does the presence of the
14 // option set the target true, or does it toggle the target? see TD in .cpp
