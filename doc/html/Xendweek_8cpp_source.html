98  // NOTE: The P-code generator works in 6-second "chunks". This implies
99  // that there are 6 seconds of X1 bitstream and 6 seconds of X2 bitstreams
100  // held in memory. To mimimize the memory footprint, these bitstreams
101  // are shared between all coders and are located in dynamically allocated
102  // buffers referenced through static pointers. THEREFORE, before any
103  // SVPCodeGen objects are instantiated, these buffers must be allocated
104  // and initialized via the following two calls. Failure to do so will
105  // resort in abnormal program termination (unless the exceptions are
106  // trapped).
107  try
108  {
109  X1Sequence::allocateMemory();
110  X2Sequence::allocateMemory();
111  }
112  catch (gpstk::Exception e)
113  {
114  printf(" Memory allocation failure.\n");
115  printf(" Xbegweek will terminate.\n");
116  return(1);
117  }
