21 /* SVNumXRef.hpp
22 *
23 * Applied Research Laboratories, The University of Texas at Austin
24 * June 2007
25 *
26 * This class implements a cross-reference between PRN numbers and NAVSTAR IDs.
27 * Since PRN numbers are re-used (historically, there's a limit of 1-32 in
28 * the PRN IDs) it's necessary provide a several->one PRN ID ->NAVSTAR mapping.
29 * Originally, the NAVSTAR->PRN ID mapping is unique. However, this is no longer
30 * true and several -> one mapping now necessary in both directions.
31 *
32 * As a result, the cross reference methods support the ability to provide a
33 * date so the method can select between multiple options.
34 *
35 * Note: Thie information is not contained anywhere in the navigation message.
36 * Therefore, it needs to come from "outside the system". The information
37 * contained in the class constructor will also need to updated with each
38 * new launch and disposal. This is a sore point, but there appears to be
39 * no easy way around it.
40 *
41 * Note: The data in the module is derived from several sources and there
42 * are some issues of interpretation.
43 * - The USNO website ftp://tycho.usno.navy.mil/pub/gps/gpsb2.txt contains
44 * a very helpful history
45 * - This class only attempts to track the relationship between PRN ID and
46 * NAVSTAR numbers. There's no guarantee a SV was operationally, broadcasting,
47 * or healthy.
48 *
49 * MEMBER METHODS
50 * SVNumXRef( ); - Constructor. Works from a fixed table that is compiled
51 * into the code.
52 * int getNAVSTAR( const int PRNID, const gpstk::CommonTime dt ) const; -
53 * Given a PRNID and a date, return the NAVSTAR number
54 * related to thsat PRNID at the date provided. May
55 * throw "NoNAVSTARNumFound" error. The date defaults to
56 * the current date if one is not provided.
57 * int getPRNID( const int NAVSTARID, const gpstk::CommonTime dt ) const; -
58 * Given a NAVSTAR Number and a date, return the corresponding
59 * PRNID related to the NAVSTAR Number on the specified date.
60 * May throw "No PRNNumberFound" error." The date defaults to
61 * the current date if one is not provided.
62 *
63 * The following four members parallel the preceding four and provide a
64 * means of determining is the specified information is available prior to
65 * making a "get" call. This is useful if you want to avoid writing
66 * "try/catch" blocks.
67 * bool PRNIDavailable( const int NAVSTARID, const gpstk::CommonTime dt ) const;
68 * bool NAVSTARIDAvailable( const int PRNID, const gpstk::CommonTime dt ) const;
69 *
70 *
71 * bool NAVSTARIDActive( const int NAVSTARID, const gpstk::CommonTime dt ) const; -
72 * Given a NAVSTAR ID and a date, return true if the specified NAVSTAR ID was
73 * active on the date provided. The date defaults to the current date if one
74 * is not provided.
75 *
76 */
