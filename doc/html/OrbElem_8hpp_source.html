222  // the bounds between which the data in OrbElem are valid.
223  // IS-GPS-200, 705, and -800 are not specific regarding the
224  // inclusion of the boundary conditions, but it is recommended
225  // that algorithms that consider validity should treat the
226  // bounds as (beginValid, endValid]. This is because beginValid
227  // is tied to the beginning of the first transmission of the data
228  // and the data will require some seconds (at least 18 seconds
229  // in the case of legacy GPS navigation message data) to be
230  // transmitted. (See the appropriate loadData( ) function in
231  // the relevant OrbElem descendents for details on how beginValid
